# clock v0.2.4

`iclock.html` は、ブラウザでローカル実行できるデジタル時計・タイマー・予定時刻表示です。HTML、CSS、JavaScriptを1ファイルに含めており、サーバー、npm、ビルド、CDN、外部API、外部フォント、外部ライブラリは不要です。

## 起動方法

1. リポジトリ直下の `iclock.html` を現在のChromium系ブラウザで直接開きます。
2. 音声読み上げを使う場合は、画面右上の「音声を有効化」を一度クリックします。ブラウザの自動再生制限により、初回はユーザー操作が必要です。

## 時計モード

* 主表示は横幅が変わりにくい等幅数字で `HH:MM` を大きく表示し、秒は右下に小さく表示します。
* 日付は補助情報として小さく表示します。
* 時計表示と時計読み上げは端末時刻を基準にします。

## タイマーモード

* 設定パネルで「残り時間（分）」と「開始時刻」を設定します。
* 開始時刻を設定すると、端末時刻がその時刻を通過したときにタイマーを自動開始します。
* 開始、一時停止、再開、リセットを操作できます。
* Spaceで開始／一時停止／再開、Rでリセット、Escapeで設定パネルを閉じられます（入力欄編集中を除く）。
* 残り時間は通常 `MM:SS`、1時間以上では `HH:MM:SS` で表示します。負数表示にはしません。


## 予定時刻モード

「予定時刻」モードでは、会議や研修などの進行確認向けに、登録した予定のうち現在時刻に対して次に来る予定と残り時間を大きく表示します。予定は最大10件まで登録でき、有効な予定だけを時刻順に判定します。同じ時刻の予定が複数ある場合は、設定画面の登録順を維持します。

設定パネルで各予定の有効／無効、時、分、名称、読み上げ有効／無効、読み上げ文章を編集できます。「予定を追加」で行を追加し、「上へ」「下へ」「削除」で登録順を調整できます。

次の予定がある場合は、直前予定の超過表示よりも次の予定を優先し、残り時間を時計・タイマーに近い大きさで画面中央に「あと `MM:SS`」または1時間以上で「あと `HH:MM:SS`」と表示します。超過表示は当日の次の予定がない場合だけ行い、最後の予定から初期値10分の超過表示時間内は「超過中」と明示し、文字、枠、色を組み合わせて色だけに依存しない形で表示します。最後の予定の超過表示時間も過ぎた場合は「本日の予定は終了しました」と表示します。

予定時刻の判定は端末時刻を基準にします。予定時刻モードの残り時間、超過表示、読み上げ判定、日付変更判定は同じ端末時刻で扱います。初期状態では予定は空なので、設定から予定を追加してください。

予定ごとの読み上げは、既存のアプリ内音声キューを利用します。同じ日の同じ予定はページ再読込後も原則として1回だけ読み上げ、日付が変わると読み上げ済み状態をリセットします。タブ非アクティブや処理遅延から復帰した場合も、前回表示時刻と今回表示時刻の間で予定時刻を通過したかを判定します。

## 読み上げ設定

ブラウザ標準の Speech Synthesis API を使用します。外部音声サービスや外部通信は使いません。音声の種類、再生可否、発音品質はブラウザやOS環境に依存します。

* 時計読み上げは最大5件です。有効、時、分、秒、文章を設定できます。同じ日の同じ条件は1回だけ発火します。設定変更、日付変更、読み上げ状態リセットで再判定できます。
* タイマー読み上げは最大5件です。有効、残り分数、文章を設定できます。同じタイマー実行中は同じ条件を1回だけ発火します。
* 読み上げは再生中を含めて最大5件のアプリ内キューで順番に処理します。リセットや設定変更など、古い発話が不要になる操作ではキューと現在発話をキャンセルします。タイマー計測は読み上げ完了を待たず継続します。

## 時間管理方式

タイマーは `setInterval` の呼び出し回数を数えて減算しません。開始・再開時の `performance.now()` と、一時停止までの経過秒数から毎回残り時間を計算します。画面更新間隔と時間計算は分離しているため、描画遅延や非アクティブタブからの復帰時も累積ずれを抑えます。

読み上げ閾値は、前回残り時間と今回残り時間の間で指定分数（内部では秒換算した閾値）を通過したかで判定します。0分の読み上げは先に発火済みにしてから、短い非同期遅延後に終了状態へ移動します。これにより、0分メッセージの取りこぼしを防ぎます。

## テーマと保存

ライト／ダークテーマを手動切替できます。保存時は主要な数値入力の範囲をJavaScript側でも検証します。設定は `localStorage` にバージョン番号付きJSONとして保存されます。保存対象は、最後のモード、テーマ、時計読み上げ、タイマー読み上げ、タイマー残り時間、開始時刻、予定一覧、予定超過表示時間、予定読み上げ、読み上げ済み日付状態、音声速度です。旧設定に時刻補正値が残っている場合は無視され、次回保存時に引き継がれません。壊れた保存値がある場合は初期値に戻して起動します。「設定を初期化」で保存内容を初期化できます。


## GitHub Pages

GitHub Pages上でも公開確認用に表示できます。公開先URLは `https://yantkys.github.io/clock/` を想定しています。

公開内容は `main` ブランチの `iclock.html` からGitHub Actionsで自動生成される `_site/index.html` です。アプリ本体の唯一の正本はリポジトリ直下の `iclock.html` で、`index.html` はリポジトリでは管理せず、公開用成果物としてのみ生成します。

初回利用時は、GitHubの `Settings` → `Pages` → `Build and deployment` → `Source` を `GitHub Actions` に設定してください。設定を自動化したい場合は、Pagesの書き込み権限を持つPersonal Access TokenをリポジトリSecret `PAGES_TOKEN` として登録すると、workflow内の `actions/configure-pages` がPages有効化を試行します。Pagesが未設定で `PAGES_TOKEN` もない場合、デプロイ時にGitHub Pages未有効化エラーになるため、先にSource設定を確認してください。

正式なローカル利用方法は、引き続きリポジトリ直下の `iclock.html` をブラウザで直接開く方式です。GitHub Pages版は外部公開環境で動作するため、公開状態の表示確認用として扱ってください。

## 外部通信

このツールは外部通信を行いません。個人情報や利用情報を外部送信しません。

## 動作確認環境

* Chromium系ブラウザを主対象に実装しています。
* 開発時はローカルファイルとして開ける構成、外部依存がない構成、JavaScript構文エラーがないことを確認しています。

## 既知の制約

* 音声読み上げの実際の声、音量、対応言語、初回再生可否はブラウザとOSの Speech Synthesis 実装に依存します。
* 実行中タイマー状態の自動復元は行いません。ページ再読込後は停止状態から開始します。
